# FP Studio Startup 要件定義書

## 1. 背景

ソフトウェア開発の見積において、ファンクション・ポイント（FP）法は客観的な指標であるが、企業内でのテーラリングにより、その指標が主観的なものに陥りやすい問題点がある。そのため、自社のFP法と比較するためのIPA/IFPUGの定めるFP法に基づいたFP見積ツールの需要がある。

## 2. 課題

- **属人化:** 見積もり根拠が不明確で、担当者によって工数が大きく変わる。
- **学習コスト:** 計算ルールが複雑で、初心者にはFP法での見積もりは難易度が高い。
- **作業性:** 既存のFP見積ツールは、FP法の「難易度マトリクス」を都度参照して計算するのに手間がかかる。
- **データの散逸:** 過去の見積もりデータがExcelファイルなどで散らばり、再利用しにくい。

## 3. 業務要件

- **標準化:** IPA/IFPUG準拠の計算ロジックに基づき、誰でも同じ基準で見積もれるようにする。
- **簡略化:** 複雑な係数計算（VAF等）はいったん考慮せず、機能の抽出とカウント（UFP）に特化して見積もりスピードを上げる。
- **資産化:** 見積もりデータをローカルDBに保存し、いつでも参照・修正できるようにする。

## 4. 機能要件

- **プロジェクト管理機能:**
  - プロジェクトの新規作成、保存、一覧表示、削除。
- **FP計測機能（メイン）:**
  - **Function Type選択:** EI, EO, EQ, ILF, EIFの5種から選択。
  - **数値入力:** DET（項目数）およびFTR/RET（参照ファイル数）の入力。
  - **自動難易度判定:** 入力された数値から「Low / Average / High」を自動決定。
  - **ポイント算出:** IPA標準の重み係数を適用して、機能ごとのFPを算出。
- **集計・出力機能:**
  - プロジェクト全体の合計FP（UFP）をリアルタイムに集計・表示。
- **設定管理機能:**
  - デフォルトの生産性（例：1.0人日/FP）を設定し、総工数を概算。

## 5. 非機能要件

- **操作性:** 1つの機能を10秒以内に追加できるような、直感的で軽快なUI。
- **オフライン動作:** 秘匿性の高い見積もりデータを扱うため、インターネットを介さずローカル完結で動作する。
- **可搬性:** Windows/Macを問わずデスクトップアプリとして配布・実行可能。
- **データ保存:** ローカルDBを使用し、アプリを閉じてもデータが保持されること。

## 6. 技術要件

- **フレームワーク:**
  - Electron-Vite
  - Tailwind CSS v3
  - Vitest
- **ライブラリ:**
  - React
  - Drizzle ORM
- **データベース:**
  - SQLite

## 7. 申し送り事項

- **将来的な拡張:** 必要に応じて、IPAの14項目による補正係数（VAF）や、プロジェクト特性係数を「掛け算」できる機能を追加検討する。
- **エクスポート:** 現状はアプリ内閲覧のみだが、将来的にCSVやExcelでの見積書出力機能が必要になる可能性がある。
- **マスター化:** 現在はIPA標準値を固定としているが、企業独自の重みテーブルに差し替えられる「マスター管理画面」の追加を検討する。
